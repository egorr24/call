# 🚀 Настройка Railway с PostgreSQL

## 📋 Что нужно сделать:

### 1. **Добавить PostgreSQL в Railway**
1. Зайдите в ваш проект на Railway: https://railway.app/project/your-project-id
2. Нажмите **"+ New"** → **"Database"** → **"Add PostgreSQL"**
3. Railway автоматически создаст базу данных и установит переменную `DATABASE_URL`

### 2. **Проверить переменные окружения**
В разделе **Variables** вашего проекта должны быть:
```
DATABASE_URL=postgresql://postgres:password@host:port/database
JWT_SECRET=flux-messenger-super-secret-jwt-key-2024-railway
NODE_ENV=production
PORT=3000
```

### 3. **Деплой обновленного кода**
1. Убедитесь что все файлы закоммичены в Git
2. Пушните изменения: `git push origin main`
3. Railway автоматически задеплоит обновления

## ✅ **Что добавлено в код:**

### **Новые API endpoints:**
- `GET /api/users/search?q=query` - поиск пользователей
- `GET /api/users/recommended` - рекомендуемые пользователи  
- `PUT /api/profile/update` - обновление профиля
- `GET /api/messages/:chatId` - получение сообщений чата

### **Подключение к PostgreSQL:**
- Автоматическое подключение через `DATABASE_URL`
- Fallback к памяти если база недоступна
- SSL подключение для Railway
- Автоматическая синхронизация моделей

### **Переменные окружения:**
- Загрузка через `dotenv`
- Файл `.env` для локальной разработки
- Поддержка Railway переменных

## 🔧 **Как проверить что работает:**

### **В логах Railway должно быть:**
```
✅ Подключение к PostgreSQL установлено
✅ Модели базы данных синхронизированы
✅ Используем PostgreSQL базу данных
🚀 Flux Messenger запущен на порту 3000
📊 База данных: PostgreSQL
```

### **Если база не подключилась:**
```
⚠️ DATABASE_URL не найден, работаем без базы данных
❌ Ошибка подключения к базе данных: [error]
⚠️ Переключаемся на режим без базы данных
📊 База данных: Memory (fallback)
```

```bash

```

1. Откройте Railway Dashboard
2. Перейдите в **Deployments**
3. Кликните на последний деплой
4. Смотрите **Build Logs** и **Deploy Logs**

## 📝 **Структура базы данных:**

### **Таблицы:**
- `users` - пользователи
- `chats` - чаты (приватные и групповые)
- `chat_participants` - участники чатов
- `messages` - сообщения
- `files` - загруженные файлы

### **Связи:**
- User → ChatParticipant (1:N)
- Chat → ChatParticipant (1:N)
- Chat → Message (1:N)
- User → Message (1:N)
- User → File (1:N)

## 🎉 **Результат:**
После настройки мессенджер будет работать с постоянной базой данных PostgreSQL на Railway, все данные будут сохраняться между перезапусками!
