# 🚀 Awesome Messenger

Современный мессенджер с видеозвонками, регистрацией и крутым интерфейсом!

## ✨ Возможности

### 🔐 Аутентификация
- Регистрация новых пользователей
- Авторизация с JWT токенами
- Безопасное хранение паролей (bcrypt)
- Автоматический вход при повторном посещении

### 💬 Мессенджер
- Реальное время через WebSocket
- Личные чаты между пользователями
- История сообщений
- Статусы "в сети/не в сети"
- Поиск по чатам и пользователям
- Уведомления о новых сообщениях

### 📹 Видеозвонки
- HD видеозвонки через WebRTC
- Аудиозвонки
- Демонстрация экрана
- Управление камерой и микрофоном
- Работает через NAT/файрволы (TURN серверы)
- Таймер звонков

### 🎨 Интерфейс
- Современный Material Design
- Адаптивный дизайн (мобильные устройства)
- Темная тема для видеозвонков
- Анимации и переходы
- Аватары пользователей (автогенерация)

## 🚀 Деплой на Railway

1. Загрузи все файлы из папки `messenger-app/`
2. Railway автоматически установит зависимости
3. Приложение запустится на порту из переменной PORT
4. Получишь ссылку типа `https://твой-проект.railway.app`

## 🛠 Технологии

### Backend
- **Node.js** + Express
- **Socket.io** для WebSocket
- **JWT** для аутентификации
- **bcryptjs** для хеширования паролей
- **UUID** для генерации ID

### Frontend
- **Vanilla JavaScript** (ES6+)
- **WebRTC** для видеозвонков
- **Socket.io Client**
- **CSS3** с анимациями
- **Font Awesome** иконки

## 📱 Функционал

### Регистрация/Вход
- Валидация форм
- Проверка уникальности email/username
- Автоматические аватары
- Сохранение сессии

### Чаты
- Создание чатов с любым пользователем
- Отправка сообщений в реальном времени

- Счетчик непрочитанных сообщений

- Кнопки вызова в каждом чате

- Демонстрация экрана

## 🔧 Настройка

### Переменные окружения (опционально)
```bash
PORT=3000                    # Порт сервера
JWT_SECRET=your-secret-key   # Секретный ключ для JWT
```

### Для продакшена
- Смени JWT_SECRET на случайную строку
- Добавь MongoDB для постоянного хранения
- Настрой HTTPS
- Добавь rate limiting
- Настрой логирование

## 📊 Хранение данных

Сейчас используется **память сервера** (Map):
- ✅ Быстро для разработки
- ❌ Данные теряются при перезапуске

Для продакшена добавь **MongoDB**:
```javascript
// Замени Map на MongoDB коллекции
const users = new Map();     // → users collection
const chats = new Map();     // → chats collection  
const messages = new Map();  // → messages collection
```

## 🐛 Известные ограничения

- Данные в памяти (перезапуск = потеря данных)
- Нет групповых чатов
- Нет отправки файлов
- Простая валидация форм
- Один сервер (нет кластера)
