# 🚀 Flux Messenger

Современный мессенджер нового поколения с видеозвонками, чатами и файлообменом.

## ✨ Возможности

### 🔐 Аутентификация
- Регистрация новых пользователей
- Авторизация с JWT токенами
- Безопасное хранение паролей (bcrypt)
- Автоматический вход при повторном посещении

### 💬 Мессенджер
- Реальное время через WebSocket
- Личные чаты между пользователями
- История сообщений
- Статусы "в сети/не в сети"
- Поиск по чатам и пользователям
- Уведомления о новых сообщениях

### 📹 Видеозвонки
- HD видеозвонки через WebRTC
- Аудиозвонки
- Демонстрация экрана
- Управление камерой и микрофоном
- Работает через NAT/файрволы (TURN серверы)
- Таймер звонков

### 📁 Файлообмен
- Отправка фото и видео
- Документы (PDF, DOC, TXT)
- Архивы (ZIP, RAR)
- Файлы до 50MB
- Превью изображений

### 🎨 Интерфейс
- Современный черно-серый дизайн
- Адаптивный дизайн (мобильные устройства)
- Темная тема
- Анимации и переходы
- Настройки профиля с аватарами

## 🚀 Деплой на Railway

### 1. Создание проекта
1. Зарегистрируйся на [Railway.app](https://railway.app)
2. Создай новый проект
3. Загрузи файлы из папки `messenger-app/`

### 2. Добавление PostgreSQL базы данных
1. В проекте Railway нажми **"+ New"**
2. Выбери **"Database"** → **"PostgreSQL"**
3. Railway автоматически создаст базу и установит переменную `DATABASE_URL`

### 3. Настройка переменных окружения (опционально)
```bash
JWT_SECRET=your-super-secret-jwt-key-here
NODE_ENV=production
```

### 4. Деплой
Railway автоматически:
- Установит зависимости (`npm install`)
- Создаст таблицы в PostgreSQL
- Запустит сервер (`npm start`)
- Даст тебе URL типа `https://flux-messenger.railway.app`

### 🚀 Режимы работы:

### ✅ С PostgreSQL (Railway - рекомендуется)
```
✅ Подключение к PostgreSQL установлено
✅ Модели базы данных синхронизированы
🚀 Flux Messenger запущен на порту 3000
📊 База данных: PostgreSQL
```
**Данные сохраняются навсегда!**

### ⚠️ Fallback режим (без базы данных)
```
⚠️  DATABASE_URL не найден, работаем без базы данных
⚠️  Используем временное хранилище в памяти
⚠️  Данные будут потеряны при перезагрузке сервера
🚀 Flux Messenger запущен на порту 3000
📊 База данных: Memory (fallback)
```
**Данные теряются при перезагрузке!**

## 💾 База данных

### PostgreSQL схема:
- **users** - пользователи
- **chats** - чаты
- **chat_participants** - участники чатов
- **messages** - сообщения
- **files** - загруженные файлы

### Автоматическая миграция:
При первом запуске Sequelize автоматически создаст все таблицы.

### Отказоустойчивость:
- Если PostgreSQL недоступен, приложение автоматически переключается на fallback режим
- Приложение не крашится, продолжает работать
- Все функции доступны, но данные временные

## 🛠 Технологии

### Backend
- **Node.js** + Express
- **Socket.io** для WebSocket
- **PostgreSQL** + Sequelize ORM (с fallback на память)
- **JWT** для аутентификации
- **bcryptjs** для хеширования паролей
- **Multer** для загрузки файлов

### Frontend
- **Vanilla JavaScript** (ES6+)
- **WebRTC** для видеозвонков
- **Socket.io Client**
- **CSS3** с анимациями
- **Font Awesome** иконки

## 🔧 Устранение неполадок

### Приложение не подключается к базе данных
- ✅ Проверь что PostgreSQL добавлен в Railway проект
- ✅ Убедись что переменная DATABASE_URL установлена
- ✅ Приложение автоматически переключится на fallback режим

### Данные теряются при перезагрузке
- ⚠️ В fallback режиме данные хранятся в памяти
- ✅ Добавь PostgreSQL в Railway для постоянного хранения
- ✅ Проверь логи подключения к базе данных

### Видеозвонки не работают
- ✅ Разреши доступ к камере/микрофону в браузере
- ✅ Используй HTTPS (обязательно для WebRTC)
- ✅ Проверь что оба пользователя онлайн
