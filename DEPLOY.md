# Инструкция по развертыванию Flux Messenger на Railway

## Шаг 1: Подготовка проекта

1. Убедитесь, что все файлы созданы и находятся в правильной структуре
2. Создайте Git репозиторий и загрузите код:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin YOUR_GITHUB_REPO_URL
   git push -u origin main
   ```

## Шаг 2: Настройка Railway

1. Зайдите на [railway.app](https://railway.app)
2. Создайте новый проект
3. Выберите "Deploy from GitHub repo"
4. Подключите ваш GitHub репозиторий

## Шаг 3: Добавление PostgreSQL

1. В проекте Railway нажмите "New Service"
2. Выберите "Database" → "PostgreSQL"
3. Railway автоматически создаст базу данных и предоставит DATABASE_URL

## Шаг 4: Настройка переменных окружения

В настройках вашего основного сервиса добавьте переменные:

```
NODE_ENV=production
JWT_SECRET=your_super_secret_jwt_key_here_make_it_long_and_random_at_least_32_characters
PORT=5000
```

DATABASE_URL будет добавлен автоматически при подключении PostgreSQL.

## Шаг 5: Настройка сборки

Railway использует nixpacks.toml для сборки. Файл уже настроен и включает:

- Установку Node.js 18
- Установку зависимостей для всех частей проекта
- Сборку React приложения
- Запуск сервера

## Шаг 6: Деплой

1. Сделайте commit и push в ваш репозиторий
2. Railway автоматически начнет деплой
3. Следите за логами в Railway dashboard

## Шаг 7: Проверка

После успешного деплоя:

1. Откройте URL вашего приложения (Railway предоставит его)
2. Попробуйте зарегистрироваться
3. Проверьте отправку сообщений
4. Протестируйте видеозвонки (требует HTTPS)

## Возможные проблемы и решения

### Ошибка "npm ci can only install packages when your package.json and package-lock.json are in sync"
- Решение: Используйте `npm install` вместо `npm ci`
- Файл nixpacks.toml уже настроен правильно

### Ошибка подключения к базе данных
- Проверьте, что PostgreSQL сервис запущен в Railway
- Убедитесь, что DATABASE_URL правильно настроен
- Проверьте логи сервера в Railway

### Ошибки сборки клиента
- Проверьте, что все зависимости установлены
- Убедитесь, что TypeScript файлы корректны
- Проверьте логи сборки в Railway

### Проблемы с WebRTC
- WebRTC требует HTTPS для работы в продакшене
- Railway автоматически предоставляет HTTPS
- Убедитесь, что пользователи дали разрешение на камеру/микрофон

### Проблемы с Socket.io
- Убедитесь, что CORS настроен правильно
- Проверьте, что клиент подключается к правильному URL
- В продакшене Socket.io должен использовать тот же домен

## Команды для локальной разработки

```bash
# Установка всех зависимостей
npm run install:all

# Запуск в режиме разработки
npm run dev

# Сборка для продакшена
npm run build

# Запуск продакшен версии локально
npm start
```

## Дополнительные настройки

### Кастомный домен
1. В Railway перейдите в Settings → Domains
2. Добавьте ваш домен
3. Настройте DNS записи

### Мониторинг
- Railway предоставляет встроенные метрики
- Настройте уведомления для критических ошибок
- Используйте логи для отладки

### Масштабирование
- Railway автоматически масштабирует приложение
- Для высоких нагрузок рассмотрите использование Redis для сессий
- Настройте горизонтальное масштабирование при необходимости
